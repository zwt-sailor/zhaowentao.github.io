数组理论基础

  • 数组是存放在连续内存空间上的相同类型数据的集合。
  • 数组的下标都是从0开始的。
  • 数组内存空间地址是连续的。
  • 在C++中二维数组是连续分布的。
  • 在java中，由于java是没有指针的，寻址操作完全交给虚拟机，所以看不到每一个元素的地址情况，故二维数组的每一行头结点的地址是没有规则的。

Leetcode704 二分查找

二分法的前提条件：有序数组、数组中无重复元素

1、初次思路：三个指针，分别指向首（i）、尾（j）、中间（mid），通过while循环，不断二分区间，直到找到目标元素；若找到则返回数组下标值；若找不到则返回-1；以下为我通过的代码：（不断根据测试用例，调整if分支，确保每一种可能都有出路）

2、标准答案：（2种）

第一种写法：左闭右闭

第二种写法：左闭右开

3、总结：

  • 左闭右闭：while(left <= right)属于合法区间，如[1,1]； 因为区间是左闭右闭所以[left, middle - 1]与[middle + 1, right]（middle均已经在上一轮检测过了）；
  • 左闭右开 ：while(left < right)属于合法区间，如[1,2)；（left不可能等于right）；
  • 常规思路下，在计算**(right + left)时，可能给的两个值都比较大，比如 [int.MAX -1, int.MAX]，导致(right + left) 会超出 int 的取值范围**，导致溢出；
  • 防溢出思路下，会首先计算**(left - right)**在保证两个值都是正数的情况下，不会造成超出 int 取值范围，这样在最后的加法运算中，也是在 [right, left] 的范围内。

Leetcode27 移除元素

1、初次思路：因为数组是连续的，所以通过一次循环将所要移除的元素变成-1（任何一个不在0-50范围内的数），同时利用count对移除元素进行计数，返回移除后数组的新长度就是原长度减去count；但发现题目要求移除后数组的前几个元素为原元素，故需要对非-1的数进行重排列。（❌超出时间限制)直接用一次循环，若该元素为所要移除的元素，则后面的元素往前移，同时对该位置元素再进行一次判断。（二次判断可能需要用到while循环❌超出时间限制)

2、标准答案：

第一种写法：（暴力解法：两层for循环，一个for循环遍历数组元素 ，第二个for循环更新数组。）

第二种写法：（双指针法/快慢指针法，快指针：寻找新数组的元素 ，新数组就是不含有目标元素的数组；慢指针：指向更新 新数组下标的位置）

  • 本题目的实质就是实现erase函数O(n)删除（覆盖）元素的一个过程。

二分法

  • 坚持循环不变量原则，只有在循环中坚持对区间的定义，才能清楚的把握循环中的各种细节。
  • 二分法是算法面试中的常考题！

双指针法

  • 双指针法（快慢指针法）：通过一个快指针和慢指针在一个for循环下完成两个for循环的工作。

  • 暴力解法时间复杂度：O(n^2) -> 双指针时间复杂度：O(n)

  • 数组中的元素为什么不能删除，主要是因为以下两点：

    1. 数组在内存中是连续的地址空间，不能释放单一元素，如果要释放，就是全释放（程序运行结束，回收内存栈空间）。
    2. C++中vector和array的区别一定要弄清楚，vector的底层实现是array，封装后使用更友好。

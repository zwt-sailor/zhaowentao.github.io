Databases 数据库

一般情况下，可以在应用程序上实现和管理最简单的数据库，即可以用它来存储数据和一堆用逗号分隔的值文件或CSV文件。

以数字音乐商店为示例，将有两个文件：一个用于艺术家，另一个用于专辑。艺术家可以具有姓名、年份和国家属性，而专辑具有姓名、艺术家和年份属性。

假设需要查找“Ice Cube” 的出道年份，通过’Python’方式来实现。

但是这种方式并不是最好的，存在一些问题。

  • 数据完整性

    1. 如何确保每个专辑条目的艺术家都是相同的？
    2. 如果有人用无效字符串覆盖专辑年份怎么办？
    3. 如何存储一张专辑中有多个艺术家？

  • 实现

    1. 你如何找到一个特定的记录？
    2. 如果我们现在想要创建一个使用相同数据库的新应用程序怎么办？
    3. 如果两个线程同时尝试写入同一个文件怎么办？

  • 持久性

    1. 如果在程序更新记录时机器崩溃了怎么办？
    2. 如果想在多台机器上复制数据库以获得高可用性怎么办？

基于以上问题，就出现了数据库管理系统（DBMS）来对数据进行管理操作。

数据库管理软件是一种专业的软件，允许程序在无须底层实现的情况下，对数据库中的信息进行存储和分析，是一种能够被多种应用所复用的软件。避免重复造轮子的问题。

Database Management System 数据库管理系统

  • DBMS是一种允许应用程序在数据库中存储和分析信息的软件

  • Early DBMSs：因为逻辑层和物理层之间存在紧密耦合，所以数据库应用程序很难构建和维护。其中逻辑层描述数据库具有哪些实体和属性，而物理层描述这些实体和属性的存储方式。早期，物理层是在应用程序代码中定义的，所以如果我们想更改应用程序使用的物理层，我们必须更改所有代码以匹配新的物理层。

Relational Model 关系模型

  • 数据模型：描述数据库中数据的概念集合。

  • 关系模型：是数据模型的一个示例。

  • 模式（schema）: 使用给定数据模型对特定数据集合的描述。

  • 关系：数据的一个列，是一个实体的属性之一。关系是一个无序集合，包含表示实体的属性的关系。由于关系是无序的，DBMS可以以任何方式存储它们，从而实现优化。

  • 关系型数据模型的三个要素：

  1. 关系结构（structure）：关系的定义及其结构的内容。
  2. 完整性（Integrity）：确保数据库的内容满足限制。
  3. 操纵性（manipulation）：如何访问和修改数据库的内容。

  • 主键（Primary Keys）：某一个唯一属性或一个属性组能够唯一标识一条记录。称其属性或属性组为主键。

  • 外键（Foreign Keys）：用于指定一张表中的属性必须存在于另一张表中。

Data Manipulation Languages (DMLs) 数据操作语言

从数据库中存储和检索信息的一种语言。这有两类语言：

1.过程性：查询指定DBMS应用于查找所需结果的（高级）策略。（经典例子：关系代数）

2.非程序性或声明性：查询只指定需要什么数据，而不指定如何查找数据（经典例子：关系演算）

Relational Algebra 关系代数

关系代数是检索和处理关系中元组的一组基本操作。每一个操作都接受一个或多个关系作为输入，并输出一个新的关系。

select

查询复合条件元组的一个子集。（挑选）

Projection

生成一个新的输出关系，里面包含一个给定输入关系中的指定属性。

Union

将两个关系组合成一个新的关系。（并集）

Intersection

两个关系表里共同的部分。（交集）

Difference

从第一个元素集中将第二个元素集中出现的元素过滤掉。（差集）

Product

笛卡尔积。所有可能性不重复的组合。
